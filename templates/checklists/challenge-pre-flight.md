# Challenge Posting Pre-Flight Checklist

Complete before publishing a challenge to the marketplace.

## Content Review
- [ ] All four required fields completed (Description, Overall Objective, Problem Statement, Desired Solution)
- [ ] Challenge emphasizes novel approaches and prototype framing
- [ ] Challenge focuses on operational need, not prescriptive solution
- [ ] No FAR/RFI procurement language remaining
- [ ] Action verbs used (prototype, demonstrate, validate, integrate, test)
- [ ] Challenge is broad enough to attract diverse submissions

## LOEs (if applicable)
- [ ] LOEs are severable work packages
- [ ] LOE language is consistent with challenge language
- [ ] LOEs provide appropriate specificity without over-constraining

## Submission Instructions
- [ ] Response length specified
- [ ] Required proposal components listed (company page, past performance, technical approach, deliverables, schedule, payment schedule, data rights, ROM by milestone)
- [ ] Eligibility criteria stated
- [ ] Formatting expectations clear (or noted as flexible)

## Timeline
- [ ] Challenge close date set
- [ ] Response window is reasonable (typically 30-45 days to down-select)
- [ ] Evaluation timeline communicated to government reviewers
- [ ] Down-select target date established

## Approvals
- [ ] Internal ONI review complete
- [ ] Government sponsor approved final text
- [ ] Contracting path language aligns with ONIX OTA / ACC-RI
- [ ] No classified information in public-facing content

## Go-Live
- [ ] Challenge uploaded to marketplace
- [ ] LOEs uploaded
- [ ] Government reviewers notified of posting
- [ ] Q&A / amendment log initialized
- [ ] Submission intake tracker ready
