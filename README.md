## Status Definitions (Quick Reference)

### Challenge Statuses
`Intake` → `Drafting` → `Gov Review` → `Live` → `Evaluating` → `Selection Pending` → `Vendor Coordination` → `Award In Progress` → `Executing` → `Archived`

### Contract Statuses
`SOW Finalizing` → `Agreement Finalizing` → `Fully Executed` → `Modification In Work` → `Executing` → `Closed`

### Invoice Statuses
`Anticipated` → `Vendor Invoice Received` → `ONI Review` → `Submitted to ACC-RI` → `Closed`

## Core PM Principle

> Keep the public challenge broad, prototype-focused, and centered on the operational need — not a prescriptive solution. Government can specify more detail in LOEs/SOW later.

## The PM's Value-Add

The PM's most important function is **translation**:
1. Translating government need into OTA challenge language
2. Translating reviewer feedback into actionable vendor edits
3. Translating vendor billing into ACC-RI-compliant invoice output
