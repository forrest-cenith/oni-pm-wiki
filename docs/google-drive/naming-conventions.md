# Naming Conventions

## Overview
Consistent file naming ensures documents are findable, sortable, and audit-ready. These conventions apply across all Google Drive project folders.

## Vendor Proposals / Submissions
```
[[Company Name]]_Proposal_[[Challenge Name]].pdf
```
**Example:** `Vertex Modernization and Sustainment LLC_Proposal_UAS Launcher_OUSW.pdf`

- Combine multiple files from one submission into a single PDF
- Sort submissions alphabetically by company name

## ACC-RI Documents

**Challenge Overview:**
```
Challenge Overview - [[marketplace.challenge.name]].docx
```

**Market Research Report:**
```
A02 Challenge Market Research Report (Market Intelligence) - [[marketplace.challenge.name]].docx
```

**Technical Evaluation & Award Selection:**
```
C04 Technical Evaluation and Award Selection Recommendation - [[marketplace.challenge.name]].docx
```

## Marketplace Challenge Names
Use the full marketplace challenge name as it appears on the platform:
```
[[Gov Org Name]] - [[Challenge Name]]
```
**Example:** `OUSW-R&E/PE Unmanned Aerial System (UAS) - Launcher Challenge`

## ONI Invoices
Follow the CLIN structure from the Operations Tracker. Include the billing period or milestone reference in the filename.

## Tips
- Never use special characters that break file paths (e.g., `/ \ : * ? " < > |`)
- Keep names descriptive but not excessively long
- When in doubt, match the naming pattern already in the project folder
