# Tracking Projects in Copper

> **This section is a placeholder.** Content will be expanded as Copper CRM workflows are documented and standardized.

## Overview

Copper CRM is used to track ONI projects across the challenge, contract, and invoice pipelines. This guide covers how PMs should use Copper to maintain visibility across all active work.

## Copper Pipelines

ONI uses three primary pipelines in Copper that mirror the PM workflow:

### 1. Challenge Pipeline
Tracks challenges from intake through archive.

| Stage | When to Move Here |
|-------|-------------------|
| OTA Intro Call | Initial government engagement begins |
| Drafting Challenge | Challenge is being written |
| Challenge Live | Challenge posted to marketplace |
| Evaluation | Submissions under review |
| Vendor Coordination | Selected vendor being coordinated |
| Prototype Execution | Prototype work underway |
| Production Execution | Follow-on production phase |
| Archive | Project closed and records archived |

### 2. Contracts Pipeline
Tracks contract status from SOW through execution.

| Stage | When to Move Here |
|-------|-------------------|
| SOW being finalized | Vendor proposal/SOW under revision |
| Project agreement being finalized | Package routed for execution |
| Fully Executed | All signatures complete |
| Modification being worked | Active modification in progress |
| Executing | Contract active, work underway |

### 3. Invoices Pipeline
Tracks each invoice from anticipation through closure.

| Stage | When to Move Here |
|-------|-------------------|
| Invoice Anticipated | Billing event approaching |
| Vendor Invoice Received | Invoice received from vendor |
| ONI Invoice Review | ONI validating and generating ACC-RI invoice |
| Submitted to ACC-RI | ONI invoice sent to ACC-RI |
| Closed | Invoice fully processed |

## Best Practices for Copper Usage

### Keep It Current
- Update pipeline stages in real-time as work progresses
- Don't batch updates — move records when the actual status changes
- Add notes when moving between stages to capture context

### Link Related Records
- Link challenge records to their corresponding contract records
- Link contract records to related invoice records
- This creates traceability across the full lifecycle
