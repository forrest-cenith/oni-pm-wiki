# Vendor Management Best Practices

## Core Practices
- Notify selected vendors quickly and clearly
- Use a standard post-selection package request: updated proposal, updated SOW if needed, W-9, reps and certs, timeline for return
- **Never ask for revisions without specific guidance** — this is the single most important vendor management rule
- Track document versions ruthlessly
- Confirm cost/ROM consistency every time a scope document changes

## Important Distinctions
PMs and vendors should clearly distinguish between:
1. **Marketplace submission** — Initial response to the challenge
2. **Updated proposal after selection** — Revised to align with government SOW
3. **Final SOW for agreement** — The governing scope document

## Tone Guidance
- Direct, professional, action-oriented
- State exactly what is needed to continue the award process
- Avoid ambiguous "please revise" language
- Be specific about what sections need changes and why

## Typical Vendor Pain Points
| Pain Point | How PM Can Help |
|------------|----------------|
| Not knowing whether proposal or SOW is the governing artifact | Clarify explicitly at each stage |
| Thinking a prior update closed all comments | State clearly when additional changes are needed and why |
| Confusion about government comments relayed secondhand | Provide written, specific feedback — not paraphrased verbal summaries |
| Billing against unclear or evolving scope | Align milestones and CLINs before execution begins |

## Post-Selection Document Request Checklist
Always request immediately after selection:
- [ ] Updated proposal aligned to government SOW
- [ ] Revised SOW (if vendor-authored)
- [ ] W-9
- [ ] Reps and Certs
- [ ] Any other required administrative forms
- [ ] Confirmed ROM and milestone/payment schedule alignment
