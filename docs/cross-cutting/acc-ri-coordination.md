# ACC-RI Coordination Guide

## Role of ACC-RI
ACC-RI (Army Contracting Command – Rock Island) serves as the contracting execution partner for ONI's OTA awards. Awards are made under the ONIX OTA in coordination with ACC-RI.

## What ACC-RI Expects
ACC-RI is a critical gate for package completeness and alignment. PMs should assume ACC-RI wants:
- Clean scope alignment between challenge, proposal, and SOW
- Coherent SOW/proposal pairing
- Complete vendor administrative forms (W-9, Reps and Certs)
- Consistent pricing/ROM across all documents

## Best Practices for ACC-RI Coordination
- **Send one clean package** rather than scattered iterative fragments
- **Ask ACC-RI for concrete discrepancy details** when they flag issues — don't guess
- **Log submission dates and responses** for every package sent
- Copy ACC-RI once the post-selection package matures, but **don't flood them with premature drafts**
- Save exactly what was submitted, not just the final PDF

## ACC-RI Touchpoints by Pipeline Stage
| Stage | ACC-RI Interaction |
|-------|-------------------|
| OTA Intro Call | Not deeply active; anticipate eventual award path |
| Drafting / Live | Minimal unless contracting path questions arise |
| Vendor Coordination | Receives updated SOW/proposal packages and routes for review |
| SOW Finalization | Reviews for discrepancies and completeness |
| Agreement Finalization | Routes for signature and execution |
| Fully Executed | Confirms execution |
| Modifications | Processes contract modifications |
| Invoicing | Receives ONI invoice packages for processing and payment |

## Invoice-Specific ACC-RI Coordination
ONI's invoicing model: the vendor submits an invoice to ONI, and ONI generates an invoice mapped to CLINs to submit to ACC Rock Island.

Key rules:
- Ensure CLIN mapping is correct before submission
- Include all required backup documentation
- Respond promptly to ACC-RI questions
- Track submission dates and processing status
